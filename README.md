# Avitotech Pull-Request Reviewer

## Допущения

В условии не прописаны некоторые детали, которые я решил уточнить для себя, предполагая, что:

- Пользователь может быть закреплен только к одной командe.

[//]: # (- При операции добавления пользователя в команду, если пользователь уже состоит в другой команде, он будет автоматически удален из предыдущей команды.)

- При операции добавления пользователя в команду, если имеется конфликт `user_id` операция будет буде прервана.
-
- В OpenAPI-спецификации указаны AdminToken и UserToken, однако в описании задания нет требований по авторизации. Для упрощения реализации авторизация не была реализована. При необходимости можно добавить middleware для проверки токенов из переменных окружения.
